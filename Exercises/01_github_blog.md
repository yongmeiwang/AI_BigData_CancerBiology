# Hands-On Activity 01  
## Establishing Your Scientific Web Presence

Modern scientific research is increasingly open, computational, and collaborative.  
A professional online presence allows researchers to share projects, datasets, code, and ideas transparently.

In this activity, you will create a simple personal webpage using GitHub Pages.

---

## Objectives

By completing this activity, you will:

- Create a public GitHub repository
- Understand how GitHub Pages deploys static websites
- Publish a basic webpage
- Reflect on scientific communication in the digital era

---
## Step 0 — Create a GitHub Account

1. Go to https://github.com
2. Click **Sign up**
3. Create a username and password.
4. Verify your email address.

Choose a professional username, as it may become part of your academic identity.

## Step 1 — Create Your Personal Repository:

1. Log in to GitHub.
2. Create a new repository named: yourusername.github.io

Replace `yourusername` with your actual GitHub username.

3. Set the repository to **Public**.
4. Initialize with a README file.

---

## Step 2 -Enable GitHub Pages
1. Go to **Settings → Pages**  
2. Under *Source*, select:
- Deploy from branch  
- Branch: `main`  
3. Save  

After 1-2 minutes, your site should be available at: 
https://yourusername.github.io 

At this stage, the page may look very simple.

---

## Step 3 — Choose a Theme

1. In your repository, go to **Settings → Pages**
2. Click **Choose a theme**
3. Select a clean academic theme (for example: Minima)
4. Commit the change

GitHub will automatically configure your site using Jekyll.

---

## Step 4 — Create Your First Blog Post

Inside your repository:

1. Click **Add file → Create new file**
2. Create a folder named:
_posts

Note: The post folder must be named as "_posts". 

3. Inside `_posts`, create a file named:

2026-02-25-first-post.md

(Note your posts should always use the date in YYYY-MM-DD format. Then your posts will appear on your blog)

4. Edit your first post with the following content:

layout: post

title: “My First Scientific Blog Post”

Welcome to my academic webpage.

This site was created as part of the AI & Big Data in Cancer Biology course.

In future posts, I will reflect on data-driven science, artificial intelligence, and research development.

5. Commit the file.

Wait 1–2 minutes, then refresh your site.

Your blog post should now appear on the homepage.


Submit the URL of your live webpage.
